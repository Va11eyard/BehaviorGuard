# Reproducing Paper Results

This document gives exact steps to reproduce the evaluation results reported in
`paper/behaviorguard.tex`.

---

## Environment

```bash
# Python 3.10+
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux / macOS
source .venv/bin/activate

pip install -e ".[dev]"
# or with poetry
poetry install
```

**Hardware used for paper results:** Single CPU (no GPU required for the sidecar).
Embedding inference runs on CPU; evaluation completes in under 10 minutes.

**Random seed:** 42 (set in `evaluation.py`, line 11).

---

## Datasets

The evaluation uses three datasets. Due to size, processed versions are not committed
to this repository. Run the following to download and preprocess them:

```bash
# PersonaChat (from Hugging Face)
python -c "
from datasets import load_dataset
ds = load_dataset('bavard/personachat_truecased')
# preprocessing script: see datasets/preprocess_personachat.py (forthcoming)
"
```

> **Note:** Preprocessing scripts that generate `datasets/*.json` from raw Hugging Face
> datasets will be added in a follow-up commit. The format expected by `evaluation.py`
> is documented in `datasets/FORMAT.md`.

Synthetic anomalies (account-takeover, social-engineering, prompt-injection) are
injected by `evaluation.py` at runtime — no separate download is needed.

## Running the Evaluation

```bash
# Full evaluation: BehaviorGuard + all 4 baselines + ablations + sensitivity + stats
python evaluation.py
```

Output: `full_evaluation_results.json`

Expected results (seed=42):

Mean latency: 13–17 ms per request (CPU, `all-MiniLM-L6-v2`).

## Recent changes (schema & scoring)

- `is_typical_active_time` is now derived from the user profile's
  `temporal_profile.most_active_hours_utc` with a fallback to 09:00–21:00
  only when no hours are learned (cold-start users).
- `normal_override` now averages only over enabled components
  (`semantic`, `linguistic`, `temporal`). When all components are
  disabled, the override is not applied.
- Each `metrics` dict in `full_evaluation_results.json` now includes a
  `per_class_metrics` block with precision, recall, F1, and support
  by both `anomaly_type` and `attack_phase`.
- New tests:
  - `tests/test_evaluation_fixes.py`
  - `tests/test_composite_scorer.py::test_normal_override_excludes_disabled_components`

---

## Deviations from Paper

- The paper reports results on 20 users per dataset (sampled in `evaluation.py`,
  `max_users=20`). Increasing this value will produce more stable estimates but
  requires more time.
- Latency figures are CPU-only. GPU inference of the embedding model would reduce
  latency further but is not required.
